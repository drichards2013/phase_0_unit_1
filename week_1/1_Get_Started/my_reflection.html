I learned a few things:
  • Version control is like a time machine. It allows us to go back in time in regard to changes to files. If we mess up, it's no big deal. We can return to the past. What's more is that we can see who made the change, collaborate on the same documents simultaneously and merge them at the end.

Here I'm going to explain the difference between Git and GitHub. When we save files regularly, we limit ourselves to only a single copy of the particular file. We have no other information about it. Git enables us to save our files and have access to information such as who made the change, what time it was made, where it was made, what exactly the change was, etc. GitHub simply lets people publish all their file history to the world so that other people can view, and, optionally, contribute to other peoples' code. Put simply, version control is like a timeline for a file or set of files. It allows us to track, revert to, and fast-forward to different time perids of a particualra file or set of files. GitHub helps with this because it allows us to see this timeline in a visual way. The GitHub interface also makes it easy to quickly find what we are looking for. Developers use version control for safety and accountability. Mess up? We've got a backup. Is something messed up? He did it. Right now, everything makes sense.
